# Trenex Version System (TVS)
The TVS is designed to solve multiple issues that came up in my mind.
It is structured in these four parts:
1. Major
2. Minor
3. Hotfix
4. Build (overall)
And is immediately followed by an 8 digit number that represents the build date by ddmmyyyy
For example,
1#23#7#89/06062026
Represents the first major version of Trenex, the 23rd minor version of Trenex 1, and the 7th hotfix to that minor revision, being the 89th build of Trenex ever, built on June 6th, 2026.
